# Especificación de Requerimientos Funcionales
### Funcionalidad 1: 
| Item              | Resultado                                                                                                                                                                                                                   |
|-------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Código            | RF-1                                                                                                                                                                                                                        |
| Nombre            | Registrar de manera jerárquica en el sistema                                                                                                                                                                                |
| Descripción       | El sistema debe permitir registrar la estructura académica jerárquica del curso                                                                                                                                             |
| Cómo se ejecutará | La funcionalidad se va a ejecutar por medio del patrón iterator ya que necesito que la persona pueda ITERAR sobre las distintas ubicaciones en que los archivos se van ubicando dependiendo de los datos que se deseen subir 
| Actor Principal   | Administradores académicos|
|Precondiciones||

### Funcionalidad 2:
| Item              | Resultado                                                                                                    |
|-------------------|--------------------------------------------------------------------------------------------------------------|
| Código | RF-2                                                                                                         |
| Nombre            | Guardado de archivos                                                                                         |                                                                                                                                                                                       |
| Descripción       | Los archivos pueden ser guardados en carpetas que pueden almacenar otras carpetas o que pueden estar vacias  |
| Cómo se ejecutará | Se va a ejecutar por medio del patrón composite que permite crear todas las carpetas de forma mucho mas fácil | 
|Precondiciones| Se deben crear las carpetas primero para que se puedan guardar los archivos                                  |
